# PyCharm / IntelliJ IDEA Configuration Guide

This document provides recommended settings for PyCharm and IntelliJ IDEA when working on dmx-learn.

## Python Interpreter Setup

1. **Open Settings:** `File → Settings` (Windows/Linux) or `PyCharm → Preferences` (macOS)
2. **Navigate to:** `Project: dmx-learn → Python Interpreter`
3. **Add Interpreter:**
   - Click the gear icon → `Add...`
   - Select `Poetry Environment`
   - Choose `Existing environment`
   - Point to: `.venv/bin/python`

## Code Style Configuration

### Black Formatter

1. **Install Black Plugin:**
   - `File → Settings → Plugins`
   - Search for "Black"
   - Install and restart PyCharm

2. **Configure Black:**
   - `File → Settings → Tools → Black`
   - Check `On code reformat`
   - Check `On save`
   - Path to Black: `.venv/bin/black`

### Import Sorting (isort)

1. **Install isort Plugin:**
   - `File → Settings → Plugins`
   - Search for "isort"
   - Install and restart PyCharm

2. **Configure isort:**
   - `File → Settings → Tools → isort`
   - Check `On code reformat`
   - Path to isort: `.venv/bin/isort`

### Python Code Style

1. **Navigate to:** `File → Settings → Editor → Code Style → Python`
2. **Set the following:**
   - **Tabs and Indents:**
     - Tab size: `4`
     - Indent: `4`
     - Continuation indent: `4`
   - **Wrapping and Braces:**
     - Hard wrap at: `88`
     - Visual guide: `88`
   - **Imports:**
     - Sort imports on `Code | Reformat Code`: Enabled
     - Use isort for import organization

## Linting Configuration

### Pylint

1. **Install Pylint Plugin:**
   - `File → Settings → Plugins`
   - Search for "Pylint"
   - Install and restart PyCharm

2. **Configure Pylint:**
   - `File → Settings → Pylint`
   - Path to Pylint: `.venv/bin/pylint`
   - Path to .pylintrc: `${PROJECT_DIR}/.pylintrc`
   - Check `Scan files in current project before checkin`

### Mypy

1. **Install Mypy Plugin:**
   - `File → Settings → Plugins`
   - Search for "Mypy"
   - Install and restart PyCharm

2. **Configure Mypy:**
   - `File → Settings → Mypy`
   - Mypy executable: `.venv/bin/mypy`
   - Config file: `pyproject.toml`

## Testing Configuration

### pytest

1. **Navigate to:** `File → Settings → Tools → Python Integrated Tools`
2. **Set Default test runner:** `pytest`
3. **Run Configuration:**
   - `Run → Edit Configurations...`
   - Click `+` → `Python tests → pytest`
   - Name: `All Tests`
   - Target: `tests`
   - Python interpreter: Use project interpreter
   - Working directory: `${PROJECT_DIR}`

## File Watchers (Optional)

File watchers can automatically run formatters on save.

### Black File Watcher

1. **Navigate to:** `File → Settings → Tools → File Watchers`
2. **Click** `+` → `<custom>`
3. **Configure:**
   - Name: `Black`
   - File type: `Python`
   - Scope: `Project Files`
   - Program: `.venv/bin/black`
   - Arguments: `$FilePath$`
   - Output paths: `$FilePath$`
   - Working directory: `$ProjectFileDir$`
   - **Advanced Options:**
     - Uncheck `Auto-save edited files to trigger the watcher`
     - Check `Trigger the watcher on external changes`

### isort File Watcher

1. **Navigate to:** `File → Settings → Tools → File Watchers`
2. **Click** `+` → `<custom>`
3. **Configure:**
   - Name: `isort`
   - File type: `Python`
   - Scope: `Project Files`
   - Program: `.venv/bin/isort`
   - Arguments: `$FilePath$`
   - Output paths: `$FilePath$`
   - Working directory: `$ProjectFileDir$`

## Editor Settings

### General

1. **Navigate to:** `File → Settings → Editor → General`
2. **Configure:**
   - **On Save:**
     - Check `Remove trailing spaces on: All lines`
     - Check `Ensure every saved file ends with a line break`
   - **Auto Import:**
     - Check `Show import popup`
     - Check `Add unambiguous imports on the fly`

### Inspections

1. **Navigate to:** `File → Settings → Editor → Inspections → Python`
2. **Enable the following:**
   - `PEP 8 coding style violation`
   - `PEP 8 naming convention violation`
   - `Type checker compatibility inspection` (Mypy)
   - `Missing or empty docstring`

## Version Control Integration

### Pre-commit Hooks

PyCharm should automatically detect and use pre-commit hooks. To verify:

1. **Navigate to:** `File → Settings → Version Control → Commit`
2. **Check:** `Run Git hooks` is enabled
3. **Before Commit section:** You should see pre-commit checks listed

## Keyboard Shortcuts

Useful shortcuts for code quality:

- **Reformat Code:** `Ctrl+Alt+L` (Windows/Linux) or `Cmd+Opt+L` (macOS)
- **Optimize Imports:** `Ctrl+Alt+O` (Windows/Linux) or `Ctrl+Opt+O` (macOS)
- **Run Inspections:** `Ctrl+Alt+Shift+I` (Windows/Linux) or `Cmd+Opt+Shift+I` (macOS)
- **Run Tests:** `Ctrl+Shift+F10` (Windows/Linux) or `Ctrl+Shift+R` (macOS)

## Troubleshooting

### Poetry Environment Not Detected

If PyCharm doesn't detect the Poetry environment:

1. Run `poetry install` in the terminal
2. Restart PyCharm
3. Manually add the interpreter: `.venv/bin/python`

### Formatters Not Working

If formatters don't run:

1. Verify Poetry environment is activated
2. Check that tools are installed: `poetry run which black`
3. Restart PyCharm
4. Clear caches: `File → Invalidate Caches / Restart`

### Tests Not Running

If tests fail to run:

1. Verify pytest is installed: `poetry run pytest --version`
2. Check that working directory is set to project root
3. Ensure test runner is set to pytest (not unittest)
